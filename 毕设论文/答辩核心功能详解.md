## 一、水位传感器——数据怎么传到 STM32？

### 硬件连接

```
水位传感器 AO 引脚 → STM32 PA1（ADC1 通道1）
```

传感器输出**模拟电压**（0~3.3V），水位越高电压越高。

### 软件流程（分步详解）

**第1步：ADC 初始化**（`adcx.c`）

```c
RCC_ADCCLKConfig(RCC_PCLK2_Div6);  // ADC时钟=72M/6=12MHz
ADC_RegularChannelConfig(ADC1, ADC_Channel_1, 1, ADC_SampleTime_55Cycles5);
```

- 配置 PA1 为**模拟输入**（GPIO_Mode_AIN）
- ADC 采样时间 55.5 个周期
- 用 **DMA 方式** 自动把 ADC 结果搬运到内存，不占用 CPU

**第2步：读取 ADC 值**（`water.c`）

```c
uint16_t WATER_GetData(void) {
    for (uint8_t i = 0; i < WATER_READ_TIMES; i++) {
        tempData += WATER_ADC_Read();  // 读DMA缓存的ADC值
        delay_ms(5);
    }
    return tempData / WATER_READ_TIMES;  // 返回平均值
}
```

- 连续采样 **WATER_READ_TIMES 次**（代码里是多次采样）
- 取**平均值**滤波，减少抖动
- 返回值范围：**0~4095**（12位ADC）

**第3步：换算为水位百分比**（`main.c`）

```c
sensorData.water = (ADC值 * 100) / 4095;  // 换算为 0~100%
```

- ADC值 0 → 0%（干）
- ADC值 4095 → 100%（满水）

**第4步：阈值判断**（`main.c` AutoControl）

```c
if (sensorData.water < Sensorthreshold.water_Value && driveData.Humin_Flag == 1) {
    driveData.Water_Flag = 1;   // 开启水泵
}
if (sensorData.water >= Sensorthreshold.water_Value) {
    driveData.Water_Flag = 0;   // 关闭水泵
}
```

- 水位低于设定阈值（默认80%）→ 启动水泵
- 水位达到阈值 → 停止水泵

### 答辩时这样说

> 水位传感器输出模拟电压接到 STM32 的 PA1 引脚，通过 ADC1 通道1 采集。
> ADC 是 12 位分辨率，转换结果范围 0~4095，对应 0~100% 水位。
> 我用了 DMA 方式自动搬运数据，多次采样取平均值滤波，然后在主循环里判断水位是否低于设定阈值，低于阈值就开启继电器让水泵加水，达到阈值就停止。

## 二、步进电机——怎么工作的？

### 硬件连接

```
STM32 PB8~PB11 → ULN2003驱动板 → 28BYJ-48步进电机
 PB8 → IN1（A相）
 PB9 → IN2（B相）
 PB10→ IN3（C相）
 PB11→ IN4（D相）
```

### 步进电机基本原理

28BYJ-48 是**四相五线**减速步进电机：
- **减速比**：64:1（内部齿轮减速）
- **步距角**：5.625°（电机本体，64步/圈）
- **输出轴一圈**：64 × 64 = **4096 步**
- **驱动方式**：八拍半步模式（精度更高、力矩更均匀）

### 八拍序列（`stepmotor.c`）

```c
// 正转序列（A→AB→B→BC→C→CD→D→DA→A...）
static const uint8_t motor_seq_fwd[8] = {0x01,0x03,0x02,0x06,0x04,0x0C,0x08,0x09};
// 反转序列（反向）
static const uint8_t motor_seq_rev[8] = {0x09,0x08,0x0C,0x04,0x06,0x02,0x03,0x01};
```

每个数字对应 A/B/C/D 四相的通电状态，例如：
- `0x01` = 0001 → 只有 A 相通电
- `0x03` = 0011 → A、B 相同时通电（半步）
- `0x02` = 0010 → 只有 B 相通电
- ... 8 个节拍循环构成一个完整周期

### TIM2 中断驱动（非阻塞！）

```c
// TIM2 配置：72M/(9+1)/(7198+1) ≈ 1ms 中断一次
TIM_TimeBaseStructure.TIM_Period = 7198;
TIM_TimeBaseStructure.TIM_Prescaler = 9;
```

每次中断调用 `MOTOR_StepTick()`：
```c
void MOTOR_StepTick(void) {
    if (!motor_running) return;
    const uint8_t *seq = motor_dir ? motor_seq_rev : motor_seq_fwd;
    uint8_t phase = seq[motor_step_idx];  // 取当前节拍
    // 根据相位设置ABCD四相电平
    if (phase & 0x01) MOTOR_A_HIGH; else MOTOR_A_LOW;
    // ...
    motor_step_idx = (motor_step_idx + 1) & 7;  // 节拍索引循环0~7
}
```

**关键点**：电机步进在中断里推进，主循环不阻塞，按键和通信同时正常响应。

### 防卡粮机制（`main.c`）

```
正转5圈（4096×5=20480步）→ 反转1圈（4096步）→ 循环
```

- 食物卡住时正转无法出料
- 反转把食物抖松，再正转就能正常出料
- 这个逻辑在 `Control_Manager()` 里实现

### 答辩时这样说

> 步进电机用 TIM2 定时器产生 1ms 中断，每次中断推进一步，八拍半步模式，输出轴转一圈需要 4096 步。
> 电机驱动用 ULN2003 驱动板，STM32 的 PB8 到 PB11 控制四相通电顺序。
> 为了防止食物卡住，我设计了防卡粮机制：正转5圈后如果重量没有增加，就反转1圈把食物抖松，然后再继续正转。

## 三、喂食功能——完整流程

### 三种喂食模式

| 模式 | 触发方式 | 停止条件 |
|---|---|---|
| **手动模式** | 按 KEY3 或 APP 发 MF0 | 称重增量达标 |
| **自动模式** | 主循环判断（空闲时自动喂） | 称重增量达标 |
| **定时模式** | 到达设定时间（3个定时） | 对应定时的喂食量达标 |

### 增量称重原理（核心！）

**问题**：盘里已经有食物了，怎么判断"新加了"多少？

**解决**：记录开始喂食时的重量 `start_weight`，用增量判断：

```c
if (driveData.Food_Flag && driveData.Now_Food_Flag == 1) {
    target = Sensorthreshold.weight_Value;  // 目标喂食量
    if (sensorData.weight >= start_weight + target) {  // 增量达标！
        driveData.Food_Flag = 0;   // 停止喂食
    }
}
```

举例：
- 盘里原有 30g，`start_weight = 30`
- 目标喂 50g，则停止条件是 `weight >= 30 + 50 = 80`
- 实际新加了 50g，正确！

### 持续转动模式（不是定角度！）

```c
void MOTOR_START_CONTINUOUS(uint8_t dir) {
    motor_dir = dir;
    motor_continuous = 1;   // 持续模式标志
    motor_running = 1;       // 启动（一直转）
}
```

- 电机**一直转**，不停地推料
- 直到 `Control_Manager` 判断增量达标，调用 `MOTOR_ABORT()` 停止
- 不是转固定角度就停！

### HX711 称重原理（简单说）

```
HX711（24位ADC芯片）→ SCK/DT 两线 → STM32
```

- 24位精度，比 STM32 内部 ADC（12位）精确很多
- 读数原理：SCK 产生时钟脉冲，DT 逐位输出 24 位数据
- 称重公式：`重量 = (ADC值 / 8388607.0) × 1000 × 修正系数`
- 去皮（皮重）：空盘时记录基准值，后续重量减去皮重得到净重

### 答辩时这样说

> 喂食功能支持手动、自动、定时三种模式。
> 电机采用持续转动模式，启动后一直转，同时通过 HX711 称重传感器实时监测重量。
> 我用增量称重的方式判断停止条件：记录开始喂食时的重量，当当前重量超过"初始重量+目标喂食量"时就停止电机。
> 这样可以不受盘里已有食物的影响，保证每次精准添加设定量的食物。

## 四、喂水功能——完整流程

### 硬件连接

```
STM32 某GPIO引脚 → 继电器模块 → 水泵
RELAY_ON  → 继电器吸合 → 水泵通电 → 开始加水
RELAY_OFF → 继电器断开 → 水泵断电 → 停止加水
```

### 自动控制逻辑（`AutoControl()`）

```c
void AutoControl(void) {
    // 自动补水
    if (sensorData.water < Sensorthreshold.water_Value && driveData.Humin_Flag == 1) {
        driveData.Water_Flag = 1;   // 开启水泵
    }
    if (sensorData.water >= Sensorthreshold.water_Value) {
        driveData.Water_Flag = 0;   // 关闭水泵
    }
}
```

### 主循环执行（`Control_Manager()`）

```c
if (driveData.Water_Flag) {
    RELAY_ON;    // 继电器吸合，水泵通电
} else {
    RELAY_OFF;   // 继电器断开，水泵断电
}
```

### 答辩时这样说

> 喂水功能通过继电器控制水泵实现。
> 水位传感器实时采集水位，主循环里判断：当水位低于设定阈值（默认80%）时，STM32 输出高电平吸合继电器，水泵通电开始加水；当水位达到阈值时，输出低电平断开继电器，水泵停止。
> 阈值可以通过 APP 或按键调整，范围 0~100%。

## 五、整体控制架构（老师常问）

### 主循环结构（`main.c` while循环）

```
while(1) {
    IWDG_Feed();       // 喂狗，防止程序跑飞
    SensorScan();       // 扫描传感器（DHT11、HX711、水位）
    Key_Scan();         // 按键扫描
    ESP8266_Process();  // 处理WiFi通信
    AutoControl();      // 自动控制逻辑（定时喂食+自动补水）
    ManualControl();    // 手动控制逻辑（按键/APP命令）
    Control_Manager();  // 执行控制（启动/停止电机、继电器）
    Display_Refresh();  // 刷新TFT显示屏
}
```

### 为什么这样设计？

- **非阻塞**：电机用定时器中断驱动，不阻塞主循环
- **传感器扫描**：DHT11 每2秒读一次（降频），HX711 每次循环都读
- **看门狗**：IWDG 3秒超时，程序跑飞自动复位

### 答辩时这样说

> 主循环采用非阻塞设计，各个功能模块独立运行。
> 传感器数据实时采集，自动控制逻辑在每个循环里判断并执行，
> 步进电机用 TIM2 中断驱动，主循环不会被阻塞，按键和通信都能正常响应。
> 我还加了独立看门狗，3秒超时自动复位，提高系统可靠性。

## 六、老师可能追问的准备

**Q：为什么用 HX711 而不用 STM32 内部 ADC 测重量？**
> HX711 是专门的 24 位称重 ADC，精度远高于 STM32 内部 12 位 ADC，而且内置放大电路，可以直接连接电桥称重传感器，外围电路更简单。

**Q：步进电机会不会堵转发热？**
> 我用了 TIM2 中断非阻塞驱动，电机运转时主循环正常执行，可以实时监控。另外在设定页面加了超时保护，如果电机长时间未检测到重量变化会停止，防止堵转发热。

**Q：水位传感器准确吗？**
> 水位传感器输出的是模拟电压，会受到水质和探头氧化影响。我在代码里做了多次采样取平均的滤波处理，实际使用精度可以满足宠物喂食站的补水需求。如果需要更高精度，可以换成超声波水位传感器。

**Q：喂水和喂食会同时工作吗？**
> 不会冲突。喂水是继电器控制水泵，喂食是步进电机控制，两个完全独立的硬件通道，可以同时进行。实际代码中 `Water_Flag` 和 `Food_Flag` 是独立标志位，互不干扰。
